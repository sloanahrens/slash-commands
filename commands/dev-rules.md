# Dev Rules

Things I often have to remind you about. Continue with your task, but remember these:

---

## Know Where You Are

**Run `pwd` before file/path operations.** Don't assume current directory.

- Use absolute paths from workspace
- Stay within your configured base_path/code_path
- Respect worktree isolation (`.trees/` are separate git environments)

---

## No Attribution

**NEVER add Claude/Anthropic attribution** to commits or PRs:

| Don't | Why |
|-------|-----|
| `Co-Authored-By: Claude` | Not allowed |
| `Generated with Claude` | Not allowed |
| Any AI attribution | Not allowed |

Just write the commit message. No signatures, no co-authors, no "generated by" tags.

---

## Simple Bash Only

These are **blocked by hookify**:

| Blocked | Use Instead |
|---------|-------------|
| `cd /path && cmd` | Separate commands, or `devbot exec` |
| `$(command)` | Run command first, use result |
| `cmd1 ; cmd2` | Separate tool calls |
| `git -C /path` | `devbot` or `cd` then `git` |

**Sequential commands:** Run each separately, one tool call at a time.

**Subdirectory commands:**
- npm: `npm run build --prefix /path/to/app`
- make: `make -C /path/to/app target`

---

## Use devbot, Not Raw Git

| Use This | Not This |
|----------|----------|
| `devbot status <repo>` | `git status` |
| `devbot diff <repo>` | `git diff` |
| `devbot branch <repo>` | `git branch` |
| `devbot log <repo>` | `git log` |
| `devbot show <repo>` | `git show` |
| `devbot fetch <repo>` | `git fetch` |
| `devbot switch <repo> <branch>` | `git checkout` |

**For commands without devbot wrapper:**
```bash
devbot path my-repo        # → /full/path/to/my-repo
cd /full/path/to/my-repo
git commit                 # Then run git command
```

---

## devbot Quick Reference

**NAME commands** (take repo name):
`path`, `status`, `diff`, `branch`, `log`, `show`, `fetch`, `switch`, `check`, `make`, `todos`, `last-commit`, `config`, `deps`, `remote`, `worktrees`, `pulumi`, `deploy`, `exec`

**PATH commands** (use `devbot path` first):
`tree`, `stats`, `detect`

**Other:**
- `find-repo <owner/repo>` — GitHub lookup to local repo
- `exec <repo> <cmd>` — Run command in repo directory

---

## YAML Gotchas

- **Avoid colons in list items** — even quoted, `- echo "URL: foo"` parses wrong. Use dashes.
- **Use `|` for multiline** — avoids escaping issues
- **Validate:** `python3 -c "import yaml; print(yaml.safe_load(open('file.yml')))"`

---

## File Operations

- **NO `/tmp` files** — use `<repo>/docs/` for working files
- **Prefer editing over creating** — modify existing files when possible
- Use Read/Edit/Write tools, not cat/sed/awk
